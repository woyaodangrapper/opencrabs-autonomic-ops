# Policy Store

Keywords: OCAO, memory, policy, correction, preference, lesson, drift, Chinese output.

User-facing output must always be Chinese.

## Store When

- User corrects behavior
- User gives stable preference
- Routing mistake occurred
- Tool or approval failure repeats
- Security boundary changes
- Environment fact is durable

## Format

One line. Concrete. Actionable. No secrets.

Example:
- Route broad repo audits through `repo-audit`; do not invent language metrics before manifest detection.

## Avoid

No commit hashes, logs, transient tasks, raw debug notes, or private secrets.

## Rule

Write policy before saying it is remembered.
